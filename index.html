HTB - GoodGames

Máquina basada en una aplicación web vulnerable y escalada de privilegios a través de un backend mal configurado.

  1. Reconocimiento: Escaneo de puertos revela un servidor web (80) y un puerto para una base de datos.
  2. Inyección SQL: La página de login del portal "GoodGames" es vulnerable a inyección SQL, permitiendo bypass de autenticación.
  3. Acceso Inicial: Tras el bypass, se descubre un panel de administración con una funcionalidad de subida de archivos. Se sube una reverse shell en PHP.
  4. Escalada de Privilegios: En la máquina, se encuentra un script en el backend que se ejecuta como root. Se aprovecha una vulnerabilidad de Command Injection en ese script para obtener una shell como root.

HTB - Paper

Explotación de una vulnerabilidad en un CMS conocido y escalada a través de credenciales filtradas.

  1. Enumeración Web: Se identifica que el sitio corre una versión obsoleta de WordPress.
  2. Explotación del CMS: Usando `wpscan`, se encuentra un plugin vulnerable a Local File Inclusion (LFI). Se utiliza el LFI para leer el archivo `wp-config.php` y obtener las credenciales de la base de datos.
  3. Movimiento Lateral: La contraseña de la base de datos se reutiliza para el usuario 'wordpress' del sistema a través de SSH, obteniendo acceso.
  4. Escalada de Privilegios: El usuario tiene permisos `sudo` para ejecutar un script de backup sin contraseña. Se modifica el script para ejecutar `/bin/bash` y se obtiene acceso como root.

HTB - Netmon

Vulnerabilidad en un software de monitoreo de red y explotación de un kernel obsoleto.

  1. Descubrimiento: Un escaneo de puertos revela una interfaz web de un software de monitoreo de red (Netmon) en un puerto no estándar.
  2. RCE en Netmon: La versión del software es vulnerable a Inyección de Comandos a través de una de sus funcionalidades de diagnóstico (e.g., ping). Se utiliza para obtener una reverse shell.
  3. Enumeración Interna: Se utiliza `linpeas.sh` para enumerar el sistema. Se detecta una versión del kernel de Linux muy antigua y vulnerable.
  4. Kernel Exploit: Se busca un exploit público para la versión del kernel (como Dirty COW), se compila en la máquina víctima y se ejecuta para escalar privilegios a root.
